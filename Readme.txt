Приложенные файлы:

Gzip.exe - сама программа, сконвертированная в exe для удобства. 
(Для работы ничего дополнительного не требуется, запускается, даже если Python не установлен на PC) 
Readme.txt - Инструкция
test.txt - Фаил с объемным текстом для тестирования.
Тест работы программы - примеры отработки программы

ВАЖНО: программа работает только с файлами .txt с кодировкой "Utf-8"ы

Инструкция:

Открываем Gzip.exe, выбираем нужное нам действие (1 или 2)

1.Сжатие файла
	1.1.Метод LZ77 используется для любых объемов текста
	1.2.Если файл с текстом находится в одной директории с программой, то можем просто написать
	название файла (test) если же путь разнится, то указываем полный путь (C:\Users\Desktop\test)
	(Обязательно указываем расширение)
	1.3.Далее сохраняем фаил, как и в пункте 1.2 прописываем название нашего файла или путь
	(Также не забываем про расширение)
	После завершения работы программа закроется. (Созданный файл ищем по прописанному пути)
	1.4.Метод Хаффмана используем для объемных текстов.
	1.5.Алгоритм такой же, как и с методом LZ77, за исключение того, что при сохранении программа 
	создаст два файла
	Первый - наш сжатый текст.
	Второй - ключ. (Ключ нужен для декодирования)
2.Декодирование
	2.1.Выбираем нужный вариант. С методом LZ77 все предельно просто, следуем указаниям программы. (Указывать расширение файла обязательно (.gz))
	2.2.При декодировании методом Хаффмана потребуется ввести как текс, так и ключ, а далее стандартное сохранение.(Указывать расширение файла обязательно (.gz))
